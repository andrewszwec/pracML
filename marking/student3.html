Executive Summary and Approach

Executive Summary

The Coursera Practical Machine Learning course project focused on evaluating a dataset consisting of accelerometer, gyro, and magneto data points for a weight lifting fitness experiement. The objective of the evaluation was to train a machine learning algorithm to predict 20 outcomes.

From the analysis performed and evaluated across 5 models, it was determined that the Random Forest model predicted to an accuracy of 99%.

Data Source and original study: http://groupware.les.inf.puc-rio.br/har

Approach

The objective of this project was to develop the best model for predicting the quality of weight lifting exercises based upon the data sets provided. Developing the prediction model for the data set consisted of several key phases.

  1. Establish the Machine Learning study

  2. Execute Exploratory Data Analysis

  3. Build and Test the Model against training and validation sets

  4. Analyze and select the best fit Model

  5. Predict the outcomes of the test set

Machine Learning design

The Machine Learning algorithm was designed applying an interative approach to model training and analyzing the error rates. This iterative approach addressed the risk of over-fitting and model issues by applying a cross-validation methodology.

The goal of each iteration was to develop a separate independent model from a training set. Each iteration was evaluated based on error rate. The model with the best error rate was selected. The training and testing sets used in each iteration were derrived by splitting the overall training data using a 70:30 ratio. The following outlines the steps performed within each iteration:

  • Acquisition of cross-validated training and testing sets

  • Data preprocessing and cleansing

  • Feature Development

  • Model design and build

  • Model evaluation against test set

  • Fit Analysis based on error rate

Model development comparison and selection

Model development

The model development phase was a structured stage of iterating through various models to find the best fit. The model candidates were:

  • Linear & Quadradic discriminate analysis
  • Recursive Partitioning and Regression Trees (rpart)
  • Random Forests (rf)

Each model was run with and without pre-processing. Models with initial in sample accuracies of less than 80% were dropped from further review. This was an indication that they were not a good fit based on the data.

Model Comparison and selection

The results from the models development concluded that Quadratic Discriminate Analysis, Random Forests and Stochastic Gradient Boosting were best able to be trained to predict accurate outcomes.

Due to the fact that the random forest model acquired such a high initial rating, and is able to be executed on a shorter timescale, it was selected as the only model to continue analysis.

The model selection part focused on applying the tuned model to the test data (note not the master test data, rather the test data partitioned from the master training set). The following confusion matrix resulted from this application.
